description: Structured dev workflow with sub-tasks and approval gates
---
You must break the requested task into small, self-contained sub-tasks. 
For each sub-task, follow these 4 steps strictly:

1. **Implementation**: Modify only the files necessary for this specific sub-task.
2. **Verification**:
   - Run specific unit tests associated with the changes.
   - Run code quality checks on the modified files.
3. **The Handover**: Provide a concise, descriptive Commit Message for the change.
4. **The Gate**: **STOP** and wait for my "APPROVED" before starting the next sub-task.
